1. html 의 주석은 ? : <!-- -->

3. html 의 태그로 대문자를 쓸 수 있는가? : 그렇다
4. html 요소의 속성의 역할 : "요소" 에 대한 추가적인 정보를 제공한다
5. 값으로 큰따옴표 못쓰는가 ? : 아니다. 쓸 수 있다. 
6. 마우스 올리면 뜨는 작은 직사각형 박스를 뭐라 하는가 :  툴팁
7. 툴팁을 지정하는 속성은 무엇인가? : title
8. 문단과 단락은 같은 말인가? : 그렇다
9. p 태그를 사용하였을때 빈 줄은 어느 위치에 추가되는가?: p 태그 전, 후 모두에 추가된다
10. 강제 줄바꿈 태그는 무엇인가? : br
11. 프로그래머가 입력한 그대로 화면에 표시하는 태그는 무엇인가? : pre
12. 위의 태그를 사용하였을떄 전 후로 줄바꿈도 추가되는가?: 그렇다
13. 해당 컨텐츠를 "굵게" 표시할뿐 아니라 "중요성"까지 내포하는 태그는 무엇인가? : <strong>
14. 해당 컨텐츠를 단순 굵게 표시하는 태그는 뭔가? : b
15. <em> 의 역할은 뭔가? : 글자 기울이기 + 강조
16. <strong> 태그 와 <em> 태그는 차이점이 뭔가? : 일단 둘다 중요성을 나타내는데 em 은 기울여서, strong 은 굵은 글자로 나타난다  
17. 컴퓨터 코드의 일부분을 나타내는 태그는 무엇인가? : <code>
18. <pre> 태그는 컴ㅍ터 코드의 일부분을 나타내는 태그인가? : 아니다
19. <hr>어쩌구</hr>했을때 어떤 일이 일어나는가? : 어쩌구의 앞에만 수평선이 생긴
20. 오른쪽 태그 문자(>) 를 의미하는 특수 문자는 무엇인가? : &gt;
21. 특수문자의 앞에 공통적으로 무슨 문자가 들어가는가? : & 와 ;
23. &qt; 는 무엇의 특수 문자인가? : 그런 특수 문자없다 !! &quot 가 "큰" 따옴표의 이스케이프 문자이다 
24. 클라이언트와 서버간의 통신 규약이 무엇인가 ; http
25. 페이로드가 무엇인가: 전송되는 데이터 "자채"
26. 페이로드에는 헤더, 메타데이터 등이 함께 보내지는가? : 아니다 . 헤더, 메타데이터 같은건 페이로드에 포함되지 않는다
27. 프롯시 서버를 다른 말로 뭐라고 하는가? : "대리" 서버
28. 웹 표준 개발 기관의 이름은? : W3C . World Wide Web Consosium
29. ip 와 port 의 관계는 ? : ip 는 네트워크 상의 서버의 "주소"이고, port 는 해당 서버 내의 프로세스 구분을 위해 사용하는 "번호"이다.
30. 프록시 서버의 초기 목적은 ? >> 인터넷 속도의 향상
31. 브라우져의 목적 >> "웹서버"에서 "HTML 파일"을 읽어서 화면에 "웹페이지"를 표시하는것
32. 마크업 업어의 뜻은? >> 텍스트에 태그를 붙여서 문서의 어디에 해당하는지 기술한 
33. HTML4 는 뭐라고도 불리는가? >> XHTML
34. XML 이란 무엇인가? : 문서의 구조를 설명하는 양식
35. xml 은 왜 쓰는가? >> (1) 사용자가 자신만의 태그를 정의하여 데이ㅓㅌ의 구조를 정의할 수 있는 "유연성" 제공. 그러니까 문서의 구조를 표현하기 위해서. (2) 데이터를 저장하고 전송하는 형식으로 사용한다. 

36. DTD 란 무었인가: Data Type Defintion 으로, 해당 문서의 타입을 정의하는 것이다. HTML 에서는 <!DOCTYPE html> 이 이에 해당한다
37. DOM 은 무엇인가: Document Object Model 로, HTML 문서를 브라우져가 이해할 수 있게 Tree 구조로 만든 것. 객체화 한 것이다.JS 로 이를 동적 수정이 가능하다.
38. 웹앱이 무엇인가? : 하나의 버전으로 모든 플랫폼에서 실행 가능하게 한 앱이다.
39. 특정 플랫폼 맞춤 제작으로 만든 앱은 무엇이라고 하는가? >> Native app
40. 리스트에서 똥글뱅이를 뭐라 하는가? >> 불릿

41. 속성에 값 할당 할떄 어떤 기호를 쓰는가? >> =
42. style 속성에 할당 하는 값 내부에서 어떤 기호를 쓰는가? >> :
43. 용어를 정의하기 위한 리스트를 정의하기 위해 어떤 태그를 쓰는가? >> <dl>
44. 43번의 자식 태그는 무엇인가? >> <dt> 와 <dl>
